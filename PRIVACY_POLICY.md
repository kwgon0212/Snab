# 개인정보처리방침

**Snab - Tab Management**

최종 수정일: 2025년 11월 05일

## 1. 개인정보처리방침 개요

Snab - Tab Management(이하 "본 확장 프로그램")는 사용자의 브라우저 탭을 효율적으로 관리할 수 있도록 도와주는 생산성 도구입니다. 본 확장 프로그램은 사용자의 개인정보 보호를 최우선으로 하며, 최소한의 정보만을 수집하고 처리합니다.

본 확장 프로그램은 모든 데이터를 사용자의 로컬 브라우저에만 저장하며, 외부 서버로 데이터를 전송하지 않습니다.

## 2. 수집하는 정보

### 2.1 자동 수집 정보

본 확장 프로그램은 탭 관리 기능을 제공하기 위해 다음과 같은 정보를 자동으로 수집합니다:

- **탭 정보**: 현재 열린 탭의 제목, URL, 파비콘
- **브라우저 윈도우 정보**: 열린 윈도우의 개수 및 구조
- **사용자 설정**: 스냅샷 옵션, 워크스페이스 설정

### 2.2 저장되는 정보

다음 정보는 사용자의 브라우저 로컬 스토리지(Chrome Storage)에만 저장됩니다:

- **워크스페이스 데이터**: 사용자가 생성한 워크스페이스, 그룹, 탭 정보
- **설정 정보**: 스냅샷 옵션 등 확장 프로그램 사용 설정

**중요**: 모든 데이터는 사용자의 로컬 브라우저에만 저장되며, 외부 서버로 전송되지 않습니다.

## 3. 정보 수집 및 사용 목적

### 3.1 수집 목적

- 탭 관리 기능 제공
- 워크스페이스 및 탭 그룹 생성 및 관리
- 사용자 설정 저장 및 복원

### 3.2 사용 목적

- 현재 열린 탭들을 워크스페이스 및 그룹별로 분류
- 탭 스냅샷을 통한 현재 브라우저 상태 저장
- 저장된 탭 그룹을 새 윈도우로 복원
- 사용자 맞춤형 탭 관리 환경 제공

## 4. 정보 처리 방식

### 4.1 로컬 처리

- **모든 데이터는 사용자의 브라우저 로컬 스토리지에만 저장됩니다**
- **외부 서버로 데이터를 전송하지 않습니다**
- **클라우드 동기화 기능을 제공하지 않습니다**
- **인터넷 연결 없이도 모든 기능을 사용할 수 있습니다**

### 4.2 데이터 암호화

- 브라우저의 기본 보안 기능을 활용합니다
- Chrome Storage API의 기본 보안 메커니즘을 사용합니다
- 민감한 정보는 별도로 암호화하지 않습니다 (로컬 저장만 수행)

## 5. 정보 공유 및 제공

### 5.1 제3자 제공 금지

본 확장 프로그램은 수집한 정보를 다음을 포함한 어떤 제3자에게도 제공하지 않습니다:

- 다른 서비스 제공자
- 광고 네트워크
- 분석 서비스
- 데이터 브로커

### 5.2 외부 서비스 연동

- 외부 서비스와의 연동 기능을 제공하지 않습니다
- 모든 기능은 로컬에서만 동작합니다
- 외부 API 호출을 하지 않습니다

## 6. 사용자 권리

### 6.1 데이터 접근 권리

- 사용자는 언제든지 저장된 데이터에 접근할 수 있습니다
- 브라우저 개발자 도구(Chrome DevTools)를 통해 저장된 데이터를 확인할 수 있습니다
- 데이터 내보내기 기능을 통해 백업 파일을 생성할 수 있습니다

### 6.2 데이터 삭제 권리

- 확장 프로그램 삭제 시 모든 데이터가 함께 삭제됩니다
- 개별 워크스페이스를 삭제할 수 있습니다
- 브라우저 설정에서 확장 프로그램 데이터를 수동으로 삭제할 수 있습니다

### 6.3 데이터 내보내기/가져오기

- 데이터 내보내기 기능을 통해 백업 파일(JSON)을 생성할 수 있습니다
- 백업 파일을 통해 다른 환경에서 데이터를 복원할 수 있습니다
- 모든 데이터는 사용자의 로컬 디바이스에만 저장됩니다

## 7. 쿠키 및 추적 기술

### 7.1 쿠키 사용

본 확장 프로그램은 쿠키를 사용하지 않습니다.

### 7.2 추적 기술

- 사용자 행동을 추적하지 않습니다
- 분석 도구(Google Analytics 등)를 사용하지 않습니다
- 광고 추적을 하지 않습니다
- 사용자 식별자를 생성하지 않습니다

## 8. 보안 조치

### 8.1 기술적 보안

- 브라우저의 기본 보안 기능을 활용합니다
- 로컬 스토리지의 접근 권한을 확장 프로그램에만 제한합니다
- Chrome의 샌드박스 환경에서 실행됩니다

### 8.2 관리적 보안

- 최소한의 권한만 요청합니다 (tabs, storage)
- 불필요한 데이터를 수집하지 않습니다
- 데이터 수집 및 사용 목적을 명확히 합니다

## 9. 권한 요청 사유

### 9.1 tabs 권한

**사용 목적:**

- 현재 열린 탭 정보를 읽기 위해 필요합니다
- 새 탭을 생성하고 기존 탭을 복원하기 위해 필요합니다
- 탭을 워크스페이스로 이동하거나 다른 윈도우로 이동하기 위해 필요합니다
- 탭 이벤트(생성/삭제/업데이트)를 감지하여 UI를 실시간으로 업데이트하기 위해 필요합니다

**사용 위치:**

- `src/utils/windows.ts`: 탭 정보 조회
- `src/newtab/App.tsx`: 탭 생성, 삭제, 이동
- `src/newtab/components/ui/Tab.tsx`: 탭 삭제
- `src/newtab/components/pannel/workspace/GroupCard.tsx`: 그룹에서 탭 복원
- `src/newtab/hooks/useAllWindows.tsx`: 탭 이벤트 리스너

### 9.2 storage 권한

**사용 목적:**

- 사용자 설정 및 워크스페이스 데이터를 저장하기 위해 필요합니다
- 브라우저 재시작 후에도 설정을 유지하기 위해 필요합니다
- 데이터 백업 및 복원 기능을 제공하기 위해 필요합니다

**사용 위치:**

- `src/newtab/store/workspace.ts`: 워크스페이스 데이터 저장/로드
- `src/utils/dataExport.ts`: 데이터 내보내기/가져오기
- `src/newtab/components/header/index.tsx`: 설정 저장

**중요**: 이 권한은 로컬 스토리지에만 사용되며, 외부 서버로 데이터를 전송하는 데 사용되지 않습니다.
